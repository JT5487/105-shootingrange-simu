## 👋 歡迎來到本專案

此文件協助接手的 AI 智慧體快速理解本專案的架構、邏輯與開發規範。本專案是一個整合軟硬體的**模擬射擊訓練系統**，目前已從 Python 遷移至 C++，並採用純 Web 前端進行顯示與操作。

## 🏗️ 專案架構

### 技術棧

| 層級 | 技術 | 說明 |
|------|------|------|
| **後端** | C++ | 核心追蹤邏輯，使用 OpenCV + Pylon SDK |
| **前端** | HTML/CSS/JS | 控制台與投影畫面 |
| **通訊** | HTTP (Port 8080) | 輪詢式雙向通訊 |
| **資料儲存** | JSON 檔案 | 校準參數、靶位設定、成績記錄 |

### 關鍵依賴

- **OpenCV** (C++ API) - 影像處理與光點偵測
- **Basler Pylon SDK** (C API) - 工業相機控制
- **nlohmann/json** - JSON 解析
- **cpp-httplib** - HTTP 伺服器
- **SheetJS** (前端) - Excel 匯出

## 🎯 核心功能

### 1. 射擊模式

支援四種射距模式，搭配日/夜間與晴/雨天候：

| 射距 | 靶型 | 說明 |
|------|------|------|
| 25m | 人形靶 | 近距離，靶位較大 |
| 75m | 人形靶 | |
| 175m | 人形靶 | |
| 300m | 人形靶 | 遠距離，靶位較小 |

參數儲存於 `target_settings.json`，包含：
- `scale` - 靶位縮放
- `vertical` - 垂直偏移
- `spacing` - 水平間距
- `edgePadding` - 邊緣留白
- `zeroingX/Y` - 歸零偏移

### 2. 倒靶邏輯

| 模式 | 觸發條件 |
|------|----------|
| `on_hit` | 命中即倒 |
| `on_time` | 倒計時結束自動全線倒靶 |
| `on_count` | 個別射手達指定發數後倒靶 |

- 倒靶視覺效果：CSS 3D 轉向 (rotateY 90deg)
- 只有「立靶」狀態才計分

### 3. 語音口令 SOP

語音檔案位於 `voice/` 目錄：

| 檔案 | 內容 |
|------|------|
| `全線預備.m4a` | 全線預備 |
| `左線預備.m4a` | 左線預備 |
| `右線預備.m4a` | 右線預備 |
| `六發裝子彈.m4a` | 六發裝子彈 |
| `握射預備.m4a` | 握射預備 |
| `開保險開始射擊.m4a` | 開保險，開始射擊 |
| `停止射擊.m4a` | 停止射擊 |
| `射手起立.m4a` | 射手起立 |
| `向右轉下射擊線.m4a` | 向右轉，下射擊線 |
| `下一波射手就位.m4a` | 下一波射手就位 |

### 4. 成績匯出

點擊「上傳成績」時，使用 SheetJS 自動產生 `.xlsx` 檔案，包含：
- 梯次、連隊、班別
- 編號、姓名
- 課目、中彈數

## ⚠️ 高智慧生物體注意事項

### 🔴 關鍵陷阱

1. **序號優先**  
   相機辨識使用硬體序號，不要依賴 USB Index。

2. **解析度陷阱**  
   Basler acA640-90gm 實際輸出是 **659×494**（非 640×480）。程式已改為動態偵測，**請勿改回硬編碼**。

3. **GenApi 衝突**  
   編譯時若遇到 `NODEMAP_HANDLE` 衝突，應使用 `PylonDeviceSetIntegerFeature` 系列包裹函數，避免直接引用 GenApi C 標頭檔。

4. **輪詢頻率**  
   前端以 200ms 週期輪詢。若命中遺失，檢查 `setInterval` 頻率是否足夠。

5. **日誌診斷**  
   若雷射偵測不到，觀察終端的 `[DEBUG] Max pixel` 輸出。正常雷射點數值應接近 255。

### 🟢 最佳實踐

1. 修改前先閱讀 `1226工作註記` 了解最新進度
2. 編譯後務必測試校準與命中偵測
3. 靶位參數調整請使用前端 UI，會自動儲存到 `target_settings.json`
