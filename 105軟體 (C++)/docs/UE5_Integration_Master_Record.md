# UE5 系統整合實作全紀錄 (Master Record)

## 一、 系統整合目標
本文件紀錄了將 Python 紅外線追蹤系統數據導入 UE5 的兩種主要實作方案，旨在提供教官/開發者最直覺的對接指引。

---

## 二、 核心架構：HTTP 數據橋接
無論採用哪種方案，Python 後端皆會在 **Port 8080** 提供數據 API：
- `GET /shots`：獲取最新的射擊座標 (x, y) 與射手 ID。
- `GET /state`：獲取目前的天候、模式與倒靶設定。

---

## 三、 方案一：專業藍圖對接 (VaRest 方案)
**適用場景**：長期使用、追求極致精準、無延遲感。

### 1. 準備工作
- 在 Epic Marketplace 安裝免費插件 **VaRest** 並於專案中啟用。
- 專案資料對照表請參見：`UE5_VaRest_Blueprint_Checklist.md`。

### 2. 邏輯流程
1. **數據抓取**：由 GameMode 透過計時器 (20Hz) 發送 HTTP GET 請求。
2. **數據解析**：將回傳的 JSON 字串解析為 `shooter_id`, `x`, `y`。
3. **分配判定**：將座標傳送給場景中的 `BP_Target_Base`。
4. **範圍判定**：靶子根據預設的 `MinX/MaxX/MinY/MaxY` 比例判斷是否中彈。

---

## 四、 方案二：極速快速通關 (Python 橋接方案)
**適用場景**：趕時間驗收、不熟悉 UE5 藍圖、快速跨專案測試。

### 1. 運作原理
執行一個獨立的 Python 腳本 (`ue5_bridge_fastpass.py`)，將紅外線座標轉換為 Windows 系統層級的 **滑鼠點擊**。

### 2. 雙投影幕設定關鍵
- **總寬度 (TOTAL_WIDTH)**：設定為兩台投影機橫向拼接後的解析度（例如 1920x2 = 3840）。
- **同步模擬**：程式會同步發送滑鼠點擊以及鍵盤按鍵 (1-6)，讓 UE5 即使沒有設定滑鼠點擊，也能透過「鍵盤事件」觸發靶子。

### 3. 操作步驟
1. 啟動 `ir_tracker_basler.py`。
2. 開啟 UE5 並進入 Play 模式。
3. 執行 `python ue5_bridge_fastpass.py`。

## 五、 方案優劣對照表

| 比較項目 | 方案一 (藍圖) | 方案二 (橋接器) |
| :--- | :--- | :--- |
| **整合深度** | 直接嵌入遊戲核心 | 外部模擬輸入 |
| **執行延遲** | 極低 (同步執行) | 略高 (經過 OS 轉換) |
| **開發成本** | 高 (需連線藍圖節點) | 極低 (執行腳本即可) |
| **精準度** | 最高 (不具備偏移風險) | 中 (需對應螢幕解析度) |
| **維護性** | 易於擴充複雜邏輯 | 適合快速展示效果 |

## 六、 常用變數與數值對照

### 1. 座標轉換
- **Python 端輸出**：0.0 (左) ~ 1.0 (右)。
- **UE5 內部判定**：依據靶標比例。
- **橋接器判定**：`像素 = 比例 * 總寬度`。

### 2. 射手對應按鍵 (映射表)
- 射手 1 -> 鍵盤 `1`
- 射手 2 -> 鍵盤 `2`
- ... 依此類推。

## 七、 常見問題排除 (FAQ)
- **畫面跑位**：檢查 `TOTAL_WIDTH` 是否正確設定為雙投影幕的總和，並確認 Windows 縮放比例為 100%。
- **點擊無反應**：確保 UE5 視窗是「目前作用中」的最上層視窗。
- **數據錯誤**：請先開啟瀏覽器訪問 `http://127.0.0.1:8080/shots` 確認是否有 JSON 數據產出。
