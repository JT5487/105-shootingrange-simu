# UE5 座標對接實戰指南 (X, Y 映射)

在 T91 系統中，後端傳給 UE5 的座標永遠是 **正規化座標 (Normalized Coordinates)**，範圍在 `0.0` 到 `1.0` 之間。這讓您的 UE5 內容不需要管相機解析度是多少，只需要專注於「比例」。

---

## 📐 1. 核心轉換公式

無論您的投影幕在 UE5 裡是用 **Widget (介面)** 還是 **3D Plane (平面相機)** 顯示，轉換公式如下：

*   **目標位置_X** = `投影區域寬度` * `接收到的 X`
*   **目標位置_Y** = `投影區域高度` * `接收到的 Y`

---

## 🛠️ 2. 實作步驟 (藍圖邏輯)

### 第一步：獲取數據 (VaRest)
使用 `VaRest` 插件發送 `GET http://localhost:8080/shots`。
- 從回傳的 JSON 陣列中提取 `x` 與 `y` (Float)。

### 第二步：映射到 3D 空間 (常用的方式)
假設您在專案中有一個 `BP_ShootingWall` (代表投影幕的牆壁)：
1.  **取得邊界**：取得該牆壁的 `StaticMesh` 寬度與高度。
2.  **計算擊中點**：
    - 將牆壁的「左上角」定為原點。
    - `擊中位置 = 左上角座標 + (牆壁右向量 * x * 牆壁總寬) + (牆壁下向量 * y * 牆壁總高)`。
3.  **生成彈孔**：在計算出的座標 `SpawnActor` 一個彈孔或觸發碰撞檢查。

---

## 👥 3. 多射手靶位設定 (多於 3 人)

既然您提到會有三個甚至更多靶位，建議在 UE5 採用 **「權限區域 (Zone Based)」** 判定：

### 方案 A：固定比例分區 (最簡單)
您可以根據射手人數，直接在 X 軸上切分：
- **3 人模式**：
    - 射手 1 負責：`0.0 ~ 0.33`
    - 射手 2 負責：`0.34 ~ 0.66`
    - 射手 3 負責：`0.67 ~ 1.0`
- **邏輯**：當收到 `x=0.5` 時，判斷它落在 `0.34 ~ 0.66` 區間，即判定為射手 2 的擊中。

### 方案 B：命中盒判定 (最精確)
不分區，直接讓靶子本身判斷：
1.  每個靶子 Actor 根據自己在投影幕上的位置算出自己的 `NormalizedRange (MinX, MaxX, MinY, MaxY)`。
2.  當 `(x, y)` 傳入時，每個靶子自己對比是否在範圍內。

---

## ⚠️ 4. 開發注意事項

1.  **Y 軸方向**：
    - 後端 `y=0` 是上方，`y=1` 是下方。
2.  **效能**：
    - 建議每秒輪詢次數 (Timer) 設定為 `30Hz` 或 `60Hz`。
3.  **座標對齊**：
    - 確保 UE5 的投影比例 (例如 16:9) 與實際投影幕一致，否則會有拉伸偏移。
