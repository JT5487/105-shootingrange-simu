# T91 系統修改記錄 - 2026/01/22

## 1. 啟動系統自動化與顯示優化
### 核心問題解決
- **路徑解析錯誤 (ERR_NAME_NOT_RESOLVED)**: 
  - **問題**: 由於專案路徑包含空格與括號 `(C++)`，Chrome 啟動時會誤將路徑識別為網域位址。
  - **解法**: 引入 **虛擬磁碟 (subst)** 技術。啟動時自動將專案目錄掛載為 `X:` 磁碟機，使用 `X:\index.html` 等純淨路徑開啟，徹底根除解析錯誤。
- **後端 DLL 遺失**: 
  - **問題**: `t91_tracker.exe` 啟動時找不到 OpenCV 相關 DLL (`libopencv_calib3d-412.dll`)。
  - **解法**: 在 PowerShell 啟動腳本中強制注入 MinGW64 的 `bin` 路徑至系統環境變數，確保後端能正確加載依賴庫。
- **分螢幕精確定位失效**:
  - **問題**: Chrome 視窗經常忽略座標參數，或在多螢幕環境下跳至錯誤顯示器。
  - **解法**: 
    1. 使用 **Win32 API (SetWindowPos / MoveWindow)** 強制介入視窗位置。
    2. 針對 [3號-左] [2號-中] [1號-右] 的實體佈局，硬編碼座標鎖定於 `(0,0)`, `(1920,0)`, `(3840,0)`。
    3. 加入 **EnumWindows** 搜尋 HTML 內的英文標記 (`[T91_CTRL]`, `[T91_A]`, `[T91_B]`)，大幅提升視窗識別穩定性。

### 顯示與模式優化
- **Kiosk 模式**: 更改啟動參數為 `--kiosk`，實現完全無邊框、無任務列、無標籤頁的純淨全螢幕體驗。
- **HTML 標題標籤**: 在 `index.html`、`training_screen_a.html`、`training_screen_b.html` 的 `<title>` 中加入唯一識別標記，協助啟動腳本精確捕捉視窗。

## 2. 每槍最低強度門檻 (Minimum Intensity Threshold)
- **後端實作**: 
  - 在 `IRTracker` 類別中新增 `shooter_min_intensity_` 陣列。
  - 在影像處理迴圈 (`processingLoop`) 中加入強度過濾邏輯：若偵測點面積小於該槍門檻則捨棄。
- **預設值設定**: 
  - 1 號槍（訊號較弱）：預設門檻設為 `0`。
  - 2~6 號槍：預設門檻設為 `30`。
  - 設定值持久化於 `target_settings.json` 中。
- **前端同步**: 
  - 修正控台 UI 提示文字，將「6號槍信號弱」更正為「1號槍信號弱」。
  - 確保控台載入時能正確拉取後端當前的門檻值，不再受 `localStorage` 舊資料干擾。

- **音效檔案路徑修正**:
