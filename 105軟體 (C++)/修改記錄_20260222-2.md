# T91 射擊訓練系統 修改記錄 (2026/02/22)

## 🛠 已完成項目 (Completed)

### 0. SOP 語音口令修正與後端重編譯 (2026/01/27 新增)
- **問題**：按下「訓練開始」後播放英文語音，且後端 `t91_tracker.exe` 無法啟動。
- **修正內容**：
    - **SOP 順序調整**：修正口令順序為「下一波射手就位 → 握射預備 → 六發裝子彈 → ...」。
    - **改用英文檔名音效**：避免中文檔名編碼問題，使用 `sop_01_ready.m4a`、`sop_02_prone.m4a` 等。
    - **新增 `voiceFileMap`**：中文口令 → 英文音檔對照表。
    - **新增 `voiceFileToText`**：英文音檔 → 中文口令反向對照，確保 TTS fallback 時播放中文。
    - **後端編譯修正**：REBUILD.bat 缺少 `-DUSE_PYLON` 編譯參數，導致 Pylon 相機代碼未被編譯。已修正並加入 `PATH` 環境變數設定。


### 1. SOP 語音口令修復 (SOP Voice Commands)
- **問題**：按下「訓練開始」後沒有聲音，且 SOP 分段錄音失效。
- **修正內容**：
    - 將複雜的中檔名 `射擊訓練口令腳本.m4a` 複製為 `sop.m4a`，避開路徑編碼報錯。
    - 在 `index.html` 加入實體 `<audio>` 播放標籤，並實作「相對路徑+絕對路徑」雙重備援。
    - 在啟動指令中加入 `--autoplay-policy=no-user-gesture-required`，強制解鎖 Chrome 的自動播放限制。
    - 修正 SOP 錄音路徑，從 `../voice/` 改為 `./voice/` 以適應虛擬磁碟 `X:` 環境。

### 2. 系統「一鍵結束」功能 (System Shutdown)
- **問題**：需要一個快速關閉全系統的按鈕。
- **修正內容**：
    - **後端 (C++)**：在 `main.cpp` 加入 `shutdown_system` 邏輯，功能包含：強制關死所有 Chrome 快取視窗、執行 `subst X: /D` 卸載虛擬磁碟、並安全終止後端程序。
    - **前端 (HTML)**：在控台面板增加紅色的「結束系統」按鈕，並實作確認對話框與 API 呼叫。
    - **啟動腳本 (PS1)**：優化為監測後端進程，當後端關閉時，Terminal 視窗會自動清理並退出，不再需要按 Enter。
    - **重新編譯**：建立 `REBUILD.bat` 成功解決中文路徑導致的編譯失敗問題，完成後端升級。

### 3. 訓練畫面 (A/B 螢幕) 紅點與音效修復
- **問題**：投影幕無紅點顯示，也無擊發聲音。
- **修正內容**：
    - 將連線位址從 `localhost` 統一修正為 `127.0.0.1`，解決某些環境下的連線延遲。
    - 在啟動參數加入 `--allow-file-access-from-files` 與 `--disable-web-security` 打通連線權限。
    - 在 `training_screen_a/b.html` 補上遺漏的 `playGunshotEffect()` 調用。
