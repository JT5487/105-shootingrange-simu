# T91 雙相機系統 - 開發問題整理

> 本文檔記錄系統開發過程中遇到的問題及解決方案，供後續開發者及 AI 參考。

---

## 📷 GigE 相機與偵測相關問題

### 問題 1：相機無法偵測（Pylon found 0 devices）
**症狀：** 程式顯示找不到裝置，但在 PylonViewer 可見。
**解決方案：** 在 `IRTracker::initialize()` 加入延遲等待掃描，並確保 PylonViewer 已關閉。

### 問題 2：雷射點偵測干擾與濾掉雜光
**症狀：** 環境光線（陽光或強力投射燈）造成誤報命中。
**解決方案：** 
1. 實作水平式 **靈敏度（閾值）調整** 功能，允許操作者根據環境即時微調。
2. 在 `Detector.hpp` 中優化斑點偵測（Blob detection）邏輯，根據雷射特性過濾形狀異常的光點。

## 🎨 UI/UX 佈局與空間配置問題

### 問題 3：27 吋螢幕無法完整顯示（需使用捲軸）
**症狀：** 功能面板過多，導致垂直高度超出 1080p 螢幕範圍，操作者需滾動頁面。
**解決方案：** 
*   **水平壓縮策略**：將原本垂直排列的單項資訊（如梯次資訊、環境設定、系統連線、倒靶模式）全部重構為單行水平佈局（Flex-row）。
*   **按鈕網格化**：射擊模式按鈕改為 2x2 網格，節省空間並放大點擊區域。
*   **校徽優化**：將校徽面板設為 `margin-top: auto` 放置於左側底部，填充剩餘空間且不干擾主控制面板。

## 🎯 射擊邏輯與倒靶機制

### 問題 4：倒靶狀態下的擊中計數錯誤
**症狀：** 靶位即便倒下（Turning），射擊仍會增加中彈數。
**解決方案：** 
*   在前端 `handleShot` 中加入狀態判定：只有當目標狀態為「立靶」時方可記錄分數。
*   同步前端與後端的倒靶配置 (`knockdownConfig`)，確保自動倒靶模式（中彈即倒、定時、發數）在前後端邏輯一致。

### 問題 5：倒靶視覺回饋不足
**症狀：** 頁面圖像僅顯示立/倒文字，缺乏直觀視覺呈現。
**解決方案：** 
*   **3D 轉向特效**：使用 CSS `transform: rotateY(90deg)` 實作轉向動畫，模擬真實「轉靶」動作。
*   使用專屬去背靶板圖 (`全身靶55x107靶板-去背.png`)。

## 📢 語音序列與自動化整合

### 問題 6：訓練終止後的後續操作不直觀
**症狀：** 停止射擊後，教官需手動尋找「成績存檔」或「重置」功能。
**解決方案：** 
*   **動態按鍵切換**：在「停止射擊」引發的完整語音口令（停止射擊 -> 射手起立 -> 下射擊線）執行完畢後，系統自動將主按鈕更換為藍色 **「上傳成績」** 與綠色 **「重新畫面」**。
*   **數據自動存檔與匯出**：點擊「上傳成績」直接對接資料庫 `save_results` 命令，並調用 `SheetJS` 函式庫在前端即時產生 `.xlsx` 報表，解決教官需手動謄寫成績的問題。

## 📦 擴充套件與技術支援

### 問題 7：如何在前端直接產出 Excel 檔案？
**解決方案**：
1. 引入 `xlsx.full.min.js` 函式庫。
2. 將 `state.hits` 與 `sessionInfo` 格式化為二維陣列 (AOAs)。
3. 使用 `XLSX.writeFile` 觸發下載。由於採純前端處理，不需額外後端 API 支援，提升了系統的離線作業能力。
